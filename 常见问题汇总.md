# 常见问题汇总

## 1. 环境配置

### 安装依赖后仍然无法运行

优先确认 Python、TensorFlow、CUDA、cuDNN 和显卡驱动版本是否匹配。本项目默认依赖 TensorFlow-GPU 2.2.0，建议使用与 `requirements.txt` 一致的版本。

### 没有使用 GPU

先确认安装的是 GPU 版本 TensorFlow，并检查 CUDA/cuDNN 是否可用。Windows 下可以用下面的命令查看显卡占用：

```bash
nvidia-smi
```

如果 TensorFlow 能正常识别 GPU，训练时通常能看到显存占用变化。

### 提示 `DLL load failed`

常见原因是 CUDA、cuDNN、TensorFlow 版本不匹配。建议重新核对版本，并确认 CUDA、cuDNN 的路径已经加入系统环境变量。

### 提示 `ModuleNotFoundError`

请在项目根目录运行脚本，例如：

```bash
python train.py
```

不要在子目录中直接运行，否则可能找不到 `nets`、`utils` 等项目内模块。

## 2. 数据集

### 数据集应该放在哪里

VOC 格式数据集默认放在：

```text
VOCdevkit/VOC2007/
```

标注文件放入：

```text
VOCdevkit/VOC2007/Annotations/
```

图片文件放入：

```text
VOCdevkit/VOC2007/JPEGImages/
```

### 运行 `voc_annotation.py` 后没有目标

请检查：

- XML 标注中的类别名是否和 `classes_path` 指向的类别文件完全一致。
- XML 文件名和图片文件名是否对应。
- `annotation_mode` 是否设置正确。
- 数据集中是否真的包含有效目标框。

### 如何训练自己的类别

编辑或新建类别文件，例如：

```text
model_data/cls_classes.txt
```

每行写一个类别名，然后同步修改 `voc_annotation.py`、`train.py`、`get_map.py` 和 `yolo.py` 中对应的 `classes_path`。

## 3. 训练

### 训练时报 shape 不匹配

通常是类别数量、模型结构或加载的权重文件不一致导致的。可以检查：

- `classes_path` 是否和当前数据集一致。
- `model_path` 是否加载了适配当前模型结构的 `.h5` 权重。
- 是否修改过 `phi`、输入尺寸或 anchors。

如果类别数量发生变化，可以继续使用预训练权重初始化主干网络，也可以将 `model_path` 设置为空字符串从零训练。

### 权重保存在哪里

训练权重默认保存在：

```text
logs/
```

如果需要把训练好的权重上传到远程仓库，请将需要保留的 `.h5` 文件复制到：

```text
model_data/
```

`model_data/` 目录已纳入版本管理。

### loss 不下降

可以重点检查：

- 数据集标注是否正确。
- 类别文件是否正确。
- 学习率是否过大。
- batch size 是否过小。
- 训练集数量是否太少。
- 预训练权重是否正确加载。

## 4. 预测

### 如何预测单张图片

运行：

```bash
python predict.py
```

然后输入图片路径，例如：

```text
img/street.jpg
```

### 如何检测视频或摄像头

在 `predict.py` 中将 `mode` 设置为 `video`。摄像头检测通常设置：

```python
video_path = 0
```

检测本地视频时，将 `video_path` 改为视频文件路径。

### 中文类别显示为方框

确认 `model_data/simhei.ttf` 存在，并且 `yolo.py` 中字体路径正确：

```python
model_data/simhei.ttf
```

## 5. 评估

### 如何计算 mAP

确认 `get_map.py` 和 `yolo.py` 中的 `classes_path`、`model_path` 配置正确，然后运行：

```bash
python get_map.py
```

输出结果会保存在：

```text
map_out/
```

### mAP 很低

常见原因包括：

- 训练轮数不足。
- 类别文件和测试集标注类别不一致。
- 训练集和测试集分布差异过大。
- 标注框错误较多。
- 预测置信度或 NMS 阈值设置不合适。

## 6. 远程仓库提交

`model_data/` 已从 `.gitignore` 中移除，目录内文件会随项目一起提交。当前包含：

```text
coco_classes.txt
simhei.ttf
voc_classes.txt
yolo_anchors.txt
yolov7_tiny_backbone_weights.h5
yolov7_tiny_weights.h5
```

后续新增权重时，请放入 `model_data/` 并执行正常的 Git 提交流程。
